Oral Scribe

Press a hotkey, speak, and your transcribed text lands wherever you need it — the active text field, clipboard, Apple Notes, or a file.

macOS 13+ Swift 5.9 Open Source Free
↓ Download View on GitHub

Installation

Option 1 — Homebrew (recommended)
brew tap neverprepared/oral-scribe https://github.com/neverprepared/oral-scribe
brew install --cask oral-scribe

Updates automatically when you run brew upgrade.

Option 2 — Manual

Download the latest OralScribe.dmg from Releases, open it, and drag Oral Scribe to your Applications folder.

Features

🎤

Multiple Backends

Apple Speech, WhisperKit, Whisper.cpp, or OpenAI Whisper — pick the right trade-off for your workflow.

⌨️

Global Hotkey

Press once to start, once to stop. Default ⌥⇧R, fully rebindable.

🤖

LLM Post-Processing

Clean up grammar, remove filler words, summarize, or apply a custom prompt via Ollama.

📤

Flexible Output

Active text field, clipboard, Apple Notes, or append to a file.

💊

Floating Overlay

A slim pill shows the recording timer and live waveform — always visible, never in the way.

🕐

Transcript History

Every transcription is saved with timestamp and duration. Copy any entry with one click.

Transcription Backends

Backend Speed Accuracy Internet Notes
Apple Speech Fast Good Optional Live streaming, auto-stops on silence, ~60s limit
WhisperKit Slow Excellent No On-device, punctuated output, file-based
Whisper.cpp Fast – Slow Excellent No On-device, CoreML accelerated. Fast with Tiny/Base models; slower with larger models
OpenAI Whisper Fast Excellent Yes Requires API key

All backends support optional LLM post-processing via Ollama.

Requirements

  • macOS 13 Ventura or later
  • For on-device Whisper: download a model from within the app (App → Transcription)
  • For LLM post-processing: Ollama running locally with a model pulled — ollama pull llama3.2:1b is a good starting point
  • For text injection into active fields: Accessibility permission (prompted on first use)
